Getting Started with Droidspaces

Droidspaces is a lightweight, portable Linux container runtime that lets you run full Linux environments on Android, Linux, and even in Android recovery or ramdisks — with complete init system support including systemd, OpenRC, and others.

No Termux, no emulation, no dependencies. A single static binary under 400KB.

At a Glance

  • Tiny footprint — under 400KB per architecture, statically compiled against musl libc
  • Truly native — runs directly on Android and Linux from the same binary
  • Wide architecture support — aarch64, armhf, x86_64, x86, riscv64
  • Beautiful Android app — manage unlimited containers with a clean Material 3 GUI
  • Zero dependencies — no Termux, no additional packages required

Platform Requirements

Android

  • Kernel 3.10+
  • Rooted device (KernelSU, Magisk, or APatch)
  • Custom kernel with required config options (see Kernel Configuration)
  • ARM64, ARMhf, x86_64, or x86
  • Run the built-in checker: Settings → Requirements

Linux

  • Kernel 3.10+
  • Root access (not required if the daemon is running)
  • No custom kernel needed
  • ARM64, ARMhf, x86_64, x86, or RISC-V 64
  • Run sudo droidspaces check to verify

Run Your First Container

Android

  1. Open the Droidspaces app
  2. Tap + to create a new container
  3. Select a rootfs
  4. Configure settings (name, networking mode)
  5. Tap Start

Linux

sudo mkdir -p /var/lib/droidspaces/rootfs
sudo tar -xzf ubuntu-rootfs.tar.gz -C /var/lib/droidspaces/rootfs
sudo droidspaces --name=mycontainer --rootfs=/var/lib/droidspaces/rootfs start
sudo droidspaces --name=mycontainer enter
